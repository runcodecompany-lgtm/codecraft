Current Problem

A major issue has been discovered in the platform architecture.

The platform was originally built around the idea that users simply enroll in courses.

However, the final vision of Code Craft Core is different.

The platform is NOT limited to programming.

It supports multiple knowledge domains such as:

- Programming
- Artificial Intelligence
- Cyber Security
- Mathematics
- Physics
- Chemistry
- Languages
- Design
- Business
- And future learning domains

Because of that, the platform architecture must become Track-Centric instead of Course-Centric.

The primary learning unit must be:

Learning Track

instead of:

Course

---

Goal

When a new user registers:

1. Select exactly one Primary Track.
2. Select one or more Secondary Tracks.
3. Take a Placement Test for each selected track.
4. Receive an independent level for each track.
5. Create a complete Learning Profile.
6. Receive a structured Learning Path for each track.
7. Receive recommended courses based on each track level.

---

Example

User:

Ahmed

Primary Track:

Programming

Secondary Tracks:

Physics
Chemistry

Placement Results:

Programming = Intermediate

Physics = Beginner

Chemistry = Advanced

The platform must treat each track independently.
