# Code Craft Core - General Agent Rules 📜

هذا الملف يحدد القواعد العامة والأساسية للذكاء الاصطناعي (Antigravity) أثناء تطوير وصيانة منصة **Code Craft Core**. يجب الالتزام بهذه القواعد بدقة متناهية في كل ملف يتم تعديله أو إنشاؤه.

---

## 🎨 1. معايير الهوية البصرية والتصميم (UI/UX)
- **الجمال البصري المذهل (Wow Factor)**: يجب أن تكون الواجهات جذابة للغاية عند النظرة الأولى. استخدم درجات ألوان متناسقة، خلفيات متدرجة ناعمة، وعناصر زجاجية (Glassmorphic) مع تباين ممتاز.
- **التصميم المتجاوب والموحد**: يجب أن يكون التصميم متجاوباً تماماً (Responsive) مع جميع الشاشات. التزم بالمتغيرات المحددة في `app/design-tokens.css` و `app/globals.css`.
- **دعم اللغة العربية والـ RTL**: يجب التأكد من ضبط اتجاه الصفحة والنصوص بشكل صحيح (`dir="rtl"`، محاذاة النصوص `text-right` للمحتوى العربي، عكس الأيقونات عند الحاجة، استخدام الهوامش `mr-` و `ml-` أو التباعد التلقائي `space-x-reverse`).
- **تأثيرات التحويم والحركة**: أضف دائماً تأثيرات تفاعلية سلسة وحيوية للتحويم (Hover) والتركيز (Focus) باستخدام Framer Motion أو انتقالات CSS لتشجيع المستخدم على التفاعل.
- **تجنب العناصر المؤقتة (Placeholders)**: لا تستخدم مربعات فارغة أو نصوص عشوائية؛ استخدم بيانات تجريبية واقعية أو صور حقيقية تم إنشاؤه عبر أداة الذكاء الاصطناعي.

---

## ⚙️ 2. بنية البرمجة وجودة الكود (Code Quality & Next.js)
- **Next.js App Router**: التزم ببنية المجلدات الحالية. افصل بين مكونات العميل (`"use client"`) ومكونات الخادم بشكل واضح لمنع تحميل الواجهة الأمامية بأكواد غير ضرورية.
- **الأمان والتحقق الصارم**:
  - يجب التحقق من صحة جميع المدخلات في الـ Server Actions والـ API Routes باستخدام `zod`.
  - يجب فحص الصلاحيات (RBAC) في كل صفحة أو إجراء متعلق بالمعلمين أو المديرين لمنع الوصول غير المصرح به.
- **قواعد البيانات (Prisma)**:
  - عند تعديل قاعدة البيانات، تأكد من توافق النماذج وعدم كسر العلاقات الحالية.
  - استخدم الـ Transactions (`prisma.$transaction`) عند إجراء عمليات كتابة متعددة مترابطة لضمان سلامة البيانات.
- **كتابة أكواد TypeScript نظيفة**:
  - يمنع تماماً استخدام النوع `any`. عرّف الأنواع والـ Interfaces بدقة.
  - حافظ على التوثيق والتعليقات الحالية ولا تقم بحذفها إلا إذا طلبت التغيير صراحةً.

---

## 🚀 3. ممارسات الـ SEO والأداء
- **العناوين والوصف (Metadata)**: أضف دائماً كائن `metadata` أو وظيفة `generateMetadata` لكل صفحة عامة لضمان أرشفة ممتازة بمحركات البحث.
- **HTML الدلالي (Semantic HTML)**: استخدم وسوم مثل `<header>`, `<main>`, `<section>`, `<footer>`, `<article>` بدلاً من استخدام `<div>` بشكل مفرط.
- **تحسين سرعة التحميل**: استخدم الصور بصيغة WebP أو الأبعاد المحددة سلفاً، وتجنب إعادة صياغة الواجهات بشكل يسبب اهتزاز الصفحة (layout shift).
